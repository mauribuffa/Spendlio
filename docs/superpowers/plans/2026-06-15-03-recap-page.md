# Monthly Recap Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dedicated, navigable **Monthly Recap** page to `apps/web` that renders the user's `monthly_summaries` row for a chosen month — income/expense/net `Stat`s, a category breakdown with `CategoryIcon` + `ProgressBar` (share of total), and the top merchant — with a month picker (current + prior months) and a graceful `EmptyState` when the recap for that month hasn't been built yet (the API returns 404). Web-only: no backend or contract changes.

**Architecture:** Next.js App Router server component at `app/recap/page.tsx`. The month is a **`?month=` search param** (not a path segment) — see the routing decision below. The page reads `getRecap(month)` through the existing server-side data layer (`apps/web/lib/resources.ts`) wrapped in `safe()` so an unreachable API or a missing recap degrades to an empty state rather than crashing the route. Rendering reuses `@spendlio/ui` components (`Card`, `Stat`, `MoneyAmount`, `CategoryIcon`, `ProgressBar`, `EmptyState`, `Select`). A tiny **pure helper** (`recentMonths` + `percentOfTotal`) is the only logic that gets a colocated unit test (TDD); everything else is presentational. Money stays integer minor units end-to-end and is formatted only at the UI edge by `MoneyAmount`. The month picker is a small client component (`MonthPicker.tsx`) that navigates by updating the URL query.

**Tech Stack:** Next.js 15 (App Router, React 19 server components), TypeScript (strict), `@spendlio/ui` (consumed from source), `lucide-react`, Vitest (for the pure helper test).

## Routing decision (justify briefly)

**Chosen: single `app/recap/page.tsx` reading `searchParams.month`.** Rationale:

- The recap is one view parameterised by month — a query param (`/recap?month=2026-05`) models "same page, different filter" more naturally than a path segment, and matches how the rest of this app keeps things flat (`/transactions`, `/budgets`, `/insights` — no nested dynamic segments).
- A bare `/recap` with **no** param simply defaults to the current month in the component (compute `YYYY-MM`), so there is **no redirect** to wire up — unlike a `[month]` route, where `/recap` would need a redirect to `/recap/<current>`.
- The month picker becomes a trivial `router.push('/recap?month=…')`; no route table changes when months change.

We therefore do **not** create `app/recap/[month]/page.tsx`. `searchParams` is async in Next 15 — the page awaits it (matching the framework version already in this repo).

## Nav decision: Recap coexists with Insights (do NOT delete Insights)

**Recommendation: add a dedicated `/recap` route AND keep `/insights` as-is.** The existing `insights/page.tsx` mixes a *partial* recap (income/expense/net + a flat category list, current month only) with the **AI `Assistant`** client component — the assistant is the real reason that page exists. Deleting Insights would drop the assistant; stripping the recap snippet out of Insights is an optional, separately-scoped cleanup, not part of this slice.

This slice's job is the **dedicated, fuller recap** (month picker, per-category `ProgressBar` share-of-total, top merchant, empty state). The small duplication of the three summary `Stat`s between `/insights` and `/recap` is acceptable and intentional: Insights stays "assistant + at-a-glance", Recap becomes "the monthly report you navigate by month". **No change to `insights/page.tsx` in this plan.**

```ts
/**
 * Pure helpers for the recap page. No framework / DOM imports — just date-string
 * math and integer-cents arithmetic, so they unit-test cleanly.
 */

/**
 * The last `count` calendar months ending at `month` (YYYY-MM), newest first.
 * Used to populate the month picker. Pure string math — no Date timezone traps.
 */
export function recentMonths(month: string, count: number): string[] {
  const [y, m] = month.split('-').map(Number);
  const out: string[] = [];
  let year = y;
  let mon = m; // 1-12
  for (let i = 0; i < count; i++) {
    out.push(`${year}-${String(mon).padStart(2, '0')}`);
    mon -= 1;
    if (mon === 0) {
      mon = 12;
      year -= 1;
    }
  }
  return out;
}

/**
 * `part` as a whole-number percent of `total` (both integer minor units),
 * clamped to 0–100. Returns 0 for non-positive parts or a zero/negative total.
 */
export function percentOfTotal(part: number, total: number): number {
  if (part <= 0 || total <= 0) return 0;
  return Math.min(100, Math.round((part / total) * 100));
}
```

`apps/web/app/recap/MonthPicker.tsx`:
```tsx
'use client';

import { useRouter } from 'next/navigation';
import { Select } from '@spendlio/ui';

/** Human label for a YYYY-MM month, e.g. "May 2026". */
function monthLabel(month: string): string {
  const [y, m] = month.split('-').map(Number);
  return new Date(y, m - 1, 1).toLocaleDateString('en-US', { month: 'long', year: 'numeric' });
}

/**
 * Navigates the recap page between months by updating the `?month=` query.
 * `months` is newest-first (from recentMonths); `value` is the selected month.
 */
export function MonthPicker({ months, value }: { months: string[]; value: string }) {
  const router = useRouter();
  return (
    <Select
      aria-label="Recap month"
      value={value}
      options={months.map((m) => ({ value: m, label: monthLabel(m) }))}
      onChange={(e) => router.push(`/recap?month=${e.target.value}`)}
    />
  );
}
```

## Task 3: The recap page (server component)

The page: resolve the month (search param or current), build the picker months, read `getRecap(month)` via `safe()`, and render Stats + category breakdown (with share-of-total `ProgressBar`) + top merchant — or an `EmptyState` when the recap isn't built (404) and a `Notice` when the API is unreachable.

**Files:**
- Create: `apps/web/app/recap/page.tsx`

- [ ] **Step 1: Write the page**

`apps/web/app/recap/page.tsx`:
```tsx
import { Card, Stat, MoneyAmount, CategoryIcon, ProgressBar, EmptyState } from '@spendlio/ui';
import { CalendarRange } from 'lucide-react';
import { getRecap, type MonthlySummary } from '../../lib/resources';
import { safe } from '../../lib/safe';
import { PageHeader } from '../_components/PageHeader';
import { Notice } from '../_components/Notice';
import { MonthPicker } from './MonthPicker';
import { recentMonths, percentOfTotal } from './recap-utils';

/** Current calendar month as YYYY-MM (UTC, matching the rest of the app). */
function currentMonth(): string {
  return new Date().toISOString().slice(0, 7);
}

const MONTH_COUNT = 12;

export default async function RecapPage({
  searchParams,
}: {
  searchParams: Promise<{ month?: string }>;
}) {
  const { month: monthParam } = await searchParams;
  const month = monthParam ?? currentMonth();
  const months = recentMonths(currentMonth(), MONTH_COUNT);

  // 404 (recap not built yet) and an unreachable API both land in `error`; we
  // distinguish them only in copy below — either way `data` is null.
  const { data, error } = await safe<MonthlySummary | null>(() => getRecap(month), null);

  // Share-of-total uses the sum of category amounts (each is positive minor
  // units of spend in the summary currency), not totalExpense, so the bars sum
  // to ~100% even if totalExpense includes uncategorised spend.
  const categoryTotal = data ? data.byCategory.reduce((sum, c) => sum + c.amount, 0) : 0;

  return (
    <div>
      <PageHeader
        eyebrow="Monthly recap"
        title="Recap"
        action={<MonthPicker months={months} value={month} />}
      />

      {error && data === null && month === currentMonth() ? (
        <Notice tone="warn">
          The API is not reachable yet, so this recap is empty. Start the API (apps/api) and seed the
          database to see your monthly recap.
        </Notice>
      ) : null}

      {data ? (
        <>
          <div
            style={{
              display: 'grid',
              gridTemplateColumns: 'repeat(3, 1fr)',
              gap: 'var(--space-4)',
              marginBottom: 'var(--space-6)',
            }}
          >
            <Card>
              <Stat label="Income" value={<MoneyAmount amount={data.totalIncome} currency={data.currency} size="lg" color="off" />} />
            </Card>
            <Card>
              <Stat label="Expense" value={<MoneyAmount amount={-data.totalExpense} currency={data.currency} size="lg" color="off" />} />
            </Card>
            <Card>
              <Stat label="Net" value={<MoneyAmount amount={data.net} currency={data.currency} size="lg" />} />
            </Card>
          </div>

          {data.topMerchant ? (
            <Card style={{ marginBottom: 'var(--space-6)' }}>
              <Stat label="Top merchant" value={data.topMerchant} />
            </Card>
          ) : null}

          {data.byCategory.length > 0 ? (
            <Card style={{ marginBottom: 'var(--space-6)' }}>
              <h2 style={{ fontSize: 'var(--text-lg)', fontFamily: 'var(--font-display)', marginBottom: 'var(--space-4)' }}>
                Spending by category
              </h2>
              <div style={{ display: 'grid', gap: 'var(--space-4)' }}>
                {data.byCategory.map((c) => {
                  const pct = percentOfTotal(c.amount, categoryTotal);
                  return (
                    <div key={c.category} style={{ display: 'grid', gap: 'var(--space-2)' }}>
                      <div style={{ display: 'flex', alignItems: 'center', gap: 'var(--space-3)' }}>
                        <CategoryIcon category={c.category} size="sm" />
                        <span style={{ flex: 1, textTransform: 'capitalize', fontSize: 'var(--text-sm)' }}>
                          {c.category}
                        </span>
                        <span style={{ fontSize: 'var(--text-xs)', color: 'var(--color-ink-subtle)' }}>{pct}%</span>
                        <MoneyAmount amount={-c.amount} currency={data.currency} size="sm" color="off" />
                      </div>
                      <ProgressBar value={pct} max={100} />
                    </div>
                  );
                })}
              </div>
            </Card>
          ) : null}
        </>
      ) : (
        <EmptyState
          icon={<CalendarRange />}
          title="Recap not ready yet"
          message="This month's recap hasn't been built. It's generated automatically once there's activity to summarize — check back soon."
        />
      )}
    </div>
  );
}
```
