# Settle Up Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship "Settle up" (Slice D): a `POST /settlements` + `GET /settlements` API, a web data-layer wiring, and a settle-up flow on `/split` that shows who owes whom (per currency) and lets the user record a full named payment between two people — which immediately changes `getBalances()`.

**Architecture:** A new NestJS `settlements` module (`apps/api/src/settlements`) mimics the existing `splits` module exactly: `AuthGuard` injects the demo user, `CurrentUser` decorator supplies `{ id }`, `ZodPipe(CreateSettlementInput)` validates the body, and the service scopes every query by `user_id`. Recording a payment writes a `settlements` row with `status: 'settled'` so the **already-existing** netting in `SplitsService.balances()` (which counts only `status='settled'` settlements) picks it up on the next read. The web app reads server-side via `lib/resources.ts` and writes via a `'use server'` server action that calls `revalidatePath('/split')`. Money stays integer minor units end-to-end; formatting happens only at the UI edge via `@spendlio/ui`'s `MoneyAmount`.

**Tech Stack:** NestJS 11 + Drizzle (`drizzle-orm`), Zod contracts (`@spendlio/contracts`), Vitest (contracts/core only — the API has no test runner), Next.js App Router (server components + server actions), `@spendlio/ui` (Card, Avatar, MoneyAmount, Badge, Button, Select, AmountInput).

## Context the worker must know (read before starting)

- **`CreateSettlementInput` ALREADY EXISTS** — `packages/contracts/src/split.ts` lines 81–84, exported through the `split` barrel → `packages/contracts/src/index.ts`. It is:
  ```ts
  export const CreateSettlementInput = SettlementSchema.omit({
    id: true, userId: true, createdAt: true, updatedAt: true, status: true, settledAt: true,
  });
  ```
  i.e. `{ fromPersonId, toPersonId, amount (int cents), currency }`. **Do NOT recreate it.** Task 1 only *adds a confirming test* and confirms the export — no schema change.
- **`SplitsService.balances()` ALREADY NETS SETTLEMENTS** — `apps/api/src/splits/splits.service.ts` lines 95–121 query `settlements` filtered by `eq(settlements.userId, userId)` and `eq(settlements.status, 'settled')`, then push a reverse edge `{ debtorId: toPersonId, creditorId: fromPersonId, amount }`. **`packages/core/src/balances.ts` needs NO change** — the settlement→edge conversion lives in the service, not in core. There is no core code to modify for this slice; the spec's "add subtraction to core" branch does **not** apply (confirmed by reading both files).
- **Critical subtlety — status must be `'settled'` on insert.** The DB column defaults to `status='pending'` (`packages/db/src/schema/settlements.ts` line 13), and `balances()` counts **only** `'settled'` rows. `CreateSettlementInput` deliberately omits `status`. Therefore the **service** must insert `status: 'settled'` and `settledAt: new Date()`. If it relied on the default, a recorded payment would NOT change balances — defeating the slice. This is the v1 "record a (completed) payment" semantic; pending settlements are out of scope.
- **The API has no unit-test runner** (`apps/api/package.json` has only `typecheck`, no vitest). So the "API endpoint test" is satisfied by: (a) a runnable Vitest test of the **netting math** placed where a runner exists, and (b) a documented manual `curl` smoke that asserts `user_id` scoping against the running stack. Do not add a test framework to `apps/api` for this slice (that would be an unrequested dependency/pattern change — see Golden Rules / CLAUDE.md).
- **`settlements` is already exported from `@spendlio/db`** (`packages/db/src/schema/index.ts` → `export * from './settlements'`, re-exported by `packages/db/src/index.ts`). Import `{ settlements, people }` from `@spendlio/db`.
- **Run commands** with workspace filters: `pnpm --filter @spendlio/contracts test`, `pnpm --filter @spendlio/core test`, `pnpm --filter @spendlio/api typecheck`, `pnpm --filter web typecheck`.
- **Money:** amounts are integer minor units everywhere; the form takes major units and converts to cents in the server action (same pattern as `apps/web/app/transactions/actions.ts`).
- **Commit style:** conventional commits, ending every commit body with the `Co-Authored-By` trailer required by the repo.

describe('netForUser — settlements net against a debt', () => {
  it('a payment reverse-edge reduces what I owe', () => {
    // I owe alice 1000 (from a split where alice paid). A settlement where I (me)
    // pay alice 600 is modeled as a reverse edge debtor=alice creditor=me.
    const edges: Edge[] = [
      { debtorId: 'me', creditorId: 'alice', amount: 1000 }, // split: I owe alice
      { debtorId: 'alice', creditorId: 'me', amount: 600 },  // settlement: I paid alice 600

    expect(net.get('alice')).toBe(-400); // still owe 400 after paying 600
  });

  it('a full settlement clears the balance to zero', () => {
    const edges: Edge[] = [
      { debtorId: 'me', creditorId: 'bob', amount: 750 },
      { debtorId: 'bob', creditorId: 'me', amount: 750 },
    ];
    expect(netForUser(edges, 'me').get('bob')).toBe(0);
  });
});

- [ ] **Step 1: Write the service**

`apps/api/src/settlements/settlements.service.ts`:
```ts
import { BadRequestException, Inject, Injectable } from '@nestjs/common';
import { and, desc, eq, inArray } from 'drizzle-orm';
import { people, settlements } from '@spendlio/db';
import type { CreateSettlementInput } from '@spendlio/contracts';
import { DB } from '../db/db.module';

@Injectable()
export class SettlementsService {
  constructor(@Inject(DB) private db: any) {}

  /** List the user's settlements, newest first. */
  async list(userId: string) {
    const items = await this.db
      .select()
      .from(settlements)
      .where(eq(settlements.userId, userId))
      .orderBy(desc(settlements.createdAt));
    return { items, nextCursor: null };
  }

  /**
   * Record a completed payment: from one of the user's people to another.
   * Written as status='settled' (with settledAt) so SplitsService.balances()
   * — which counts only settled rows — nets it immediately. v1 records full
   * named payments only; partial/pending settlements are out of scope.
   */
  async create(userId: string, dto: CreateSettlementInput) {
    if (dto.fromPersonId === dto.toPersonId) {
      throw new BadRequestException('A payment must be between two different people.');
    }
    if (dto.amount <= 0) {
      throw new BadRequestException('Amount must be positive.');
    }

    // Both people must belong to this user (id-only payload, scoped check).
    const owned = await this.db
      .select({ id: people.id })
      .from(people)
      .where(and(eq(people.userId, userId), inArray(people.id, [dto.fromPersonId, dto.toPersonId])));
    const ownedIds = new Set(owned.map((p: { id: string }) => p.id));
    if (!ownedIds.has(dto.fromPersonId) || !ownedIds.has(dto.toPersonId)) {
      throw new BadRequestException('Both people must belong to you.');
    }

    const [row] = await this.db
      .insert(settlements)
      .values({
        userId,
        fromPersonId: dto.fromPersonId,
        toPersonId: dto.toPersonId,
        amount: dto.amount,
        currency: dto.currency,
        status: 'settled',
        settledAt: new Date(),
      })
      .returning();
    return row;
  }
}
```

## Task 5: Web — server action `settleUpAction`

**Files:**
- Create: `apps/web/app/split/actions.ts`

Mirrors `apps/web/app/transactions/actions.ts`: a `'use server'` module, a Zod form schema (strings → cents), validate against `CreateSettlementInput`, call the resource, then `revalidatePath('/split')` and `revalidatePath('/')`.

- [ ] **Step 1: Write the action**

`apps/web/app/split/actions.ts`:
```ts
'use server';

import { revalidatePath } from 'next/cache';
import { z } from 'zod';
import { CreateSettlementInput } from '@spendlio/contracts';
import { createSettlement } from '../../lib/resources';
import { ApiError } from '../../lib/api';

export interface ActionResult {
  ok: boolean;
  error?: string;
  fieldErrors?: Record<string, string[]>;
}

// The form posts strings; coerce the amount (major units typed by a human, e.g.
// "25.00") to integer cents, then validate with the same contract the API enforces.
const FormSchema = z.object({
  fromPersonId: z.string().uuid('Pick who is paying.'),
  toPersonId: z.string().uuid('Pick who is being paid.'),
  amountMajor: z.coerce.number().positive('Enter an amount greater than zero.'),
  currency: z.string().length(3),
});

export async function settleUpAction(
  _prev: ActionResult,
  formData: FormData,
): Promise<ActionResult> {
  const parsed = FormSchema.safeParse({
    fromPersonId: formData.get('fromPersonId'),
    toPersonId: formData.get('toPersonId'),
    amountMajor: formData.get('amountMajor'),
    currency: formData.get('currency'),
  });

  if (!parsed.success) {
    return { ok: false, fieldErrors: parsed.error.flatten().fieldErrors as Record<string, string[]> };
  }

  const { fromPersonId, toPersonId, amountMajor, currency } = parsed.data;
  if (fromPersonId === toPersonId) {
    return { ok: false, error: 'A payment must be between two different people.' };
  }
  const amount = Math.round(amountMajor * 100); // assumes 2-dp entry currency

  const input = CreateSettlementInput.safeParse({
    fromPersonId,
    toPersonId,
    amount,
    currency: currency.toUpperCase(),
  });
  if (!input.success) {
    return { ok: false, fieldErrors: input.error.flatten().fieldErrors as Record<string, string[]> };
  }

  try {
    await createSettlement(input.data);
  } catch (err) {
    if (err instanceof ApiError) return { ok: false, error: err.message };
    return { ok: false, error: 'Could not record the payment.' };
  }

  revalidatePath('/split');
  revalidatePath('/');
  return { ok: true };
}
```

```tsx
'use client';

import { useActionState, useEffect, useRef } from 'react';
import { Select, AmountInput, Button } from '@spendlio/ui';
import { settleUpAction, type ActionResult } from './actions';

interface Person {
  id: string;
  name: string;
}

const CURRENCIES = ['USD', 'EUR', 'GBP', 'ARS', 'BRL', 'MXN'];
const initial: ActionResult = { ok: false };

export function SettleUpForm({ people }: { people: Person[] }) {
  const [state, formAction, pending] = useActionState(settleUpAction, initial);
  const formRef = useRef<HTMLFormElement>(null);

  useEffect(() => {
    if (state.ok) formRef.current?.reset();
  }, [state.ok]);

  if (people.length < 2) {
    return (
      <p style={{ color: 'var(--color-ink-subtle)', fontSize: 'var(--text-sm)' }}>
        Add at least two people to record a payment between them.
      </p>
    );
  }

  const options = people.map((p) => ({ value: p.id, label: p.name }));

  return (
    <form ref={formRef} action={formAction} style={{ display: 'grid', gap: 'var(--space-4)' }}>
      <div style={{ display: 'grid', gap: 'var(--space-4)', gridTemplateColumns: '1fr 1fr' }}>
        <Select name="fromPersonId" label="Payer" options={options} placeholder="Who paid" defaultValue="" required />
        <Select name="toPersonId" label="Payee" options={options} placeholder="Who was paid" defaultValue="" required />
      </div>

      <div style={{ display: 'grid', gap: 'var(--space-4)', gridTemplateColumns: '2fr 1fr', alignItems: 'end' }}>
        <div style={{ display: 'flex', flexDirection: 'column', gap: '6px' }}>
          <label style={{ fontSize: 'var(--text-sm)', fontWeight: 'var(--weight-semibold)', color: 'var(--color-ink)' }}>
            Amount
          </label>
          <AmountInput name="amountMajor" size="compact" currency="$" defaultValue="" required />
        </div>
        <Select name="currency" label="Currency" options={CURRENCIES} defaultValue="USD" />
      </div>

      {state.error ? (
        <p style={{ color: 'var(--color-negative)', fontSize: 'var(--text-sm)' }}>{state.error}</p>
      ) : null}
      {state.fieldErrors
        ? Object.entries(state.fieldErrors).map(([field, msgs]) => (
            <p key={field} style={{ color: 'var(--color-negative)', fontSize: 'var(--text-sm)' }}>
              {msgs.join(' ')}
            </p>
          ))
        : null}
      {state.ok ? (
        <p style={{ color: 'var(--color-positive)', fontSize: 'var(--text-sm)' }}>Payment recorded.</p>
      ) : null}

      <div>
        <Button type="submit" disabled={pending}>
          {pending ? 'Recording…' : 'Settle up'}
        </Button>
      </div>
    </form>
  );
}
```

> **Note on `AmountInput`/`Select` as form fields:** both forward extra props to the underlying native `<input>`/`<select>` (per the foundation plan — `AmountInput` spreads `...rest` to its `<input>`, `Select` spreads to its `<select>`), so `name`/`required`/`defaultValue` reach the real control and the uncontrolled `<form action>` submission carries them. If a typecheck error shows `name` is not accepted, confirm Task-0 foundation shipped these components with `...rest` passthrough before working around it.
