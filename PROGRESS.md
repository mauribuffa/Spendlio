| 2026-06-15 | Content-addressed receipt uploads + OCR retry-status fix (ADR-026) | **Receipts now hash-addressed.** Client computes SHA-256 (`crypto.subtle`) before the direct-to-storage PUT; `presign(contentType, sha256)` derives a content-addressed, per-user key (`receipts/<userId>/<sha256>.<ext>`); `register({imageKey, sha256})` **dedups** (same hash → returns existing receipt, skips re-OCR); the **worker re-hashes** fetched bytes and fails on mismatch (integrity). `sha256` column on `receipts` (migration `0002`, nullable, `(userId,sha256)` lookup index); `ocr` payload stays id-only. Generic `files` table + partial-unique index deliberately deferred (YAGNI).
